# v1.7.0 Responsive Smoke Verification

**Date:** 2026-05-25
**Verifier:** Coding agent via Claude_Preview MCP
**Method:** Vite dev server at `http://localhost:1420` with a temporary `window.__TAURI_INTERNALS__` stub injected into `index.html` so the React UI mounts in a plain-browser context. After verification, the stub was reverted (no production code touched).

## What was checked

Five surfaces, at three widths each. Verification used the Claude_Preview accessibility snapshot tool (`preview_snapshot`), which returns the rendered ARIA tree — preferred over screenshots for structural verification per the tool docs.

Surfaces:
1. **Home** — empty-state hero ("Pick a modpack to play" + body + "Open Modpacks" CTA) + What's New card with v1.7.0 release notes
2. **Modpacks list** — page heading + HelpHint + Quick-Add code paste row + Yours/Browse outer tabs + All/Published inner tabs + 3 modpack cards (Achievement compatible, No Mods, TEST)
3. **Modpack detail** — Back button + heading + Switch/Share buttons + "Mods in this modpack" heading + empty state + Advanced disclosure
4. **Library** — "All installed mods" heading + subtitle + "Manage active modpack →" link + 5 toolbar buttons + Installed/Browse tabs + search + sort + bulk actions + 5 mod rows ("Show details for X" expand pattern)
5. **HelpDrawer** — opens via topbar `?` icon, renders "Help" title + Playing modpacks content + 8 FAQ items

Widths:
- **Wide:** 1700×1000
- **Medium:** 1280×900
- **Narrow:** 900×800

## Result

**Zero layout breakage at any width.** All primary actions remained visible; no buttons cut off, no overflow, no overlap. The 4-item sidebar (Home / Modpacks / Library / Settings) and the topbar (profile chip + Help `?` + Vanilla + Launch) held at all widths down to 900px.

The accessibility tree structure was identical across widths — the existing `gf-*` CSS classes (flex/grid with sensible wrap behavior) handle the responsive transition without any layout collapse.

## Limitations

- This was a structural verification (ARIA tree), not a pixel-level visual review. The `preview_screenshot` tool timed out on this preview due to the visibility-hidden state (Playwright throttles backgrounded tabs); the accessibility snapshot is what was used instead.
- The Tauri runtime itself (native window controls, drag region, deep-link handlers) was not exercised — the dev-server stub uses fake data.
- For a final pixel-level check before public release, a human should run the actual Tauri binary at the three widths and eyeball each surface.

## Files touched (all reverted)

- `index.html` — temporary inline `<script>` stub added then reverted.
- `.claude/launch.json` — temp config for preview_start, deleted.

Working tree is clean after the smoke; only this verification note is committed.
