# Interaction Inventory

This is the issue #157 inventory for existing user-facing behavior. Each row is a user interaction path, the test owner that proves it, and the user-visible assertion that release managers can trust.

Status meanings:

- `Automated`: owned by a command that runs under CI, `npm run qa`, or the release gate.
- `Manual`: intentionally outside the current Rust, Vitest, or WebDriver layers. Manual rows must keep a reason and review date.

Required tag classes are checked by `npm run qa:matrix`: `global-shell`, `menu`, `dialog`, `profile-flow`, `mod-library`, `settings`, `error-state`, `empty-state`, and `large-list`.
